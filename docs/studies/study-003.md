## 1. What this study is about

Study 003 is not mainly a “single AI ad prompt” lesson.  
It is a **brand-building workflow** lesson: the creator uses AI to simulate the process of building a high-value consumer brand, then turns that brand system into product visuals and advertising assets.

The important lesson for AI 영상제작 is:

> Do not start with a random video prompt.  
> Start with a brand system, generate reusable assets, then create many ads from that asset base.

---

## 2. Actual workflow observed from the SRT

### Step 1 — Brand idea and positioning

The creator frames the project as building a major brand with AI.  
The workflow starts from the idea of making a brand that could feel valuable and scalable, not from making one isolated video.

Production logic:

1. Define the brand concept.
2. Decide the product/category.
3. Create visual identity.
4. Create product assets.
5. Convert assets into ad formats.

### Step 2 — Brand identity assets

The creator moves into brand identity creation, including:

- brand name
- logo
- visual style
- color direction
- product look
- packaging / product presentation

This is important because the ads later depend on these fixed visual decisions.

### Step 3 — AI-assisted asset generation

The creator uses AI tools as an asset factory, not as a one-shot final-output machine.

Observed pattern:

- ask AI for a prompt
- generate a visual
- evaluate the result
- refine the prompt
- regenerate or edit
- keep useful assets as reusable brand material

### Step 4 — Ad generation from the brand system

Once the identity and product assets exist, the creator can produce multiple ad types:

- product ads
- social ads
- UGC-style creatives
- commercial-style visuals
- brand campaign visuals

This is the core repeatable system: **one brand system → many outputs**.

## 3. Human → AI instruction patterns

### Pattern 1 — Objective-first prompt

The human does not start with vague visual taste.  
The prompt begins with the target output: logo, product visual, ad, brand asset, etc.

Example pattern:

```text
Create / give me a prompt for [specific brand asset] for [specific brand].
```

Why it matters:
- AI receives a clear production target.
- The output can be checked against a concrete asset need.
- This prevents random “cool image” generation.

### Pattern 2 — Role delegation

The user repeatedly treats AI as a specialist, not just a text generator.

Useful roles inferred from the workflow:

- brand strategist
- logo designer
- product designer
- creative director
- ad director
- prompt engineer
- campaign planner

HARIN adaptation:

```text
Act as a luxury beauty brand creative director.
Build a campaign asset plan for SUH HARIN as an AI advertising model.
```

### Pattern 3 — Constraint-driven refinement

The workflow depends on constraints:

- brand name fixed
- category fixed
- style direction fixed
- color palette fixed
- product identity fixed
- final use case fixed

This is more useful than saying “make it better.”

HARIN adaptation:

```text
Keep HARIN's face, hair, voice, and wardrobe identity consistent.
Generate only ad concepts that can reuse the locked HARIN asset library.
```

### Pattern 4 — Asset checklist prompting

The creator’s workflow works because it resembles a checklist, not a single prompt.

Checklist pattern:

1. brand concept
2. logo
3. product
4. package
5. product render
6. ad concepts
7. ad scripts
8. final video prompts

HARIN adaptation:

```text
Before generating any Higgsfield prompt, list every required asset:
HARIN face reference, outfit, product, location, camera style, and final platform ratio.
```

### Pattern 5 — Iterative prompt improvement

The observed workflow is iterative:

```text
Prompt → output → evaluation → edited prompt → better output
```

The creator uses AI to improve the prompt itself, not only to create the final asset.

HARIN adaptation:

```text
Review this video prompt as a Higgsfield ad director.
Find what will cause inconsistency, hallucination, or weak product visibility.
Rewrite it as a tighter cut-level prompt.
```

### Pattern 6 — Brand system before campaign

The creator does not treat ads as isolated videos.  
The ads are extensions of the brand system.

HARIN adaptation:

```text
Do not create one-off HARIN reels.
Create a reusable campaign system with fixed visual rules, reusable products, and repeatable shot structures.
```

## 4. What to learn for AI 영상제작

### Lesson A — A character must become a brand system

HARIN should not be managed only as “an AI woman who appears in ads.”  
She should become a brand-controlled IP with repeatable assets.

Required HARIN library:

- face reference
- voice reference
- wardrobe sets
- body/pose references
- camera language
- tone of speech
- brand-safe product categories
- caption/hashtag style
- ad disclosure style

### Lesson B — Ads should be generated from reusable kits

Each campaign should have a kit:

```text
Campaign kit
- Product image
- HARIN outfit
- Location
- Lighting mood
- 3 core shots
- 3 hook lines
- 3 CTA variations
- 5 short captions
```

Then the same kit can produce:

- Instagram Reel
- YouTube Shorts
- TikTok-style short
- UGC version
- premium commercial version
- product close-up version

### Lesson C — Prompting should move from “video prompt” to “production system”

Bad:

```text
Make HARIN advertise this product beautifully.
```

Good:

```text
Build a 5-shot vertical ad for HARIN.
Use the locked product asset.
Shot 1: hook close-up.
Shot 2: product problem.
Shot 3: HARIN interaction.
Shot 4: product benefit.
Shot 5: CTA.
Keep face identity, hand continuity, and product position consistent.
```

### Lesson D — AI tools should be chained by production role

Recommended tool roles:

- Claude / 클챗: brand system, shot list, prompt refinement
- Higgsfield: motion/video scene generation
- image tool: product and campaign stills
- CapCut: edit, subtitles, music, platform export
- Notion / docs: asset and campaign tracking

## 6. Rules to preserve

### Rule 1 — Brand First, Campaign Second

Before making ads, define the brand system.

### Rule 2 — Asset Library Before Content

Before generating reels, create reusable assets.

### Rule 3 — One Asset System → Many Outputs

One asset library should produce many videos, not one.

### Rule 4 — Iterative Prompt Refinement

Prompt quality improves by review and revision, not by one attempt.

### Rule 5 — Checklist-Based Production

AI video production should use checklists because consistency is the main bottleneck.
