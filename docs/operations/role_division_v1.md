# AI 영상 제작 역할 분담 (v1)

> **범위:** 이 문서는 **영상 제작 프로젝트 한정** 역할 정의다. 전체 프로젝트 운영 역할 정의가 아니다.
> 전체 운영의 4주체 체계는 [00_START_HERE.md](../../00_START_HERE.md) §3 · [DL-009](decision_log.md)를 그대로 유지한다.

관련 결정: [DL-013](decision_log.md) (DL-009 4주체를 대체하지 않고, 영상 제작 한정 역할 분담을 추가)

## 목적

역할을 명확히 분리하여 결정권 충돌과 크레딧 낭비를 방지한다.

> **창작은 Director. 판단은 CEO. 정리는 PM. 반영은 Engineer.**

## 4주체 ↔ 영상 제작 역할 매핑

| 전체 운영 4주체 | 영상 제작 역할 |
|-----------------|----------------|
| CEO (코지) | CEO |
| 코비 (ChatGPT) | PM |
| 클챗 (Claude Director) | Director |
| 클로 (Claude Code) | Engineer |

---

## CEO (코지)

**역할**
- 브랜드 방향 결정
- 광고 목적 결정
- 최종 승인 (GO / NO-GO)

**하지 않는 일**
- 프롬프트 작성
- Shot 연출
- 생성 파라미터 수정

---

## Director (클챗 / Claude Director)

**역할** — 영상 제작 총괄. 영상 제작의 모든 **창작 책임**을 가진다.

- 컨셉 구체화
- 시나리오
- Shot List
- Storyboard
- Scene 구성
- 감정선 설계
- Keyframe 제작
- 프롬프트 작성
- Higgsfield 생성
- Veo 제작
- 후반 제작 전략

---

## PM (코비 / ChatGPT)

**역할** — 창작하지 않는다. Director와 CEO 사이의 프로젝트 매니저 역할만 수행한다.

- 클챗(Director) 결과 정리
- CEO 의견 정리
- 요구사항 문서화
- 작업지시서 작성
- 진행 순서 관리
- 일정 관리
- 비용 관리
- 검수 체크리스트 관리

**하지 않는 일**
- 시나리오 작성
- Shot 설계
- 프롬프트 수정
- 연출 수정
- 생성 방식 변경
- 기술적 해결책 제시

---

## Engineer (클로 / Claude Code)

**역할** — 창작·생성 결정을 하지 않는다. 승인된 내용을 **실제 저장소·문서에 반영**하는 기술 실행 주체.

- 구현
- commit
- push
- GitHub 반영
- 문서 반영
- 기술 실행

**하지 않는 일**
- 시나리오·Shot·프롬프트 등 창작
- 생성물(창작 내용) 판단·수정

> Director는 **무엇을 만들지**(창작)를 정하고, Engineer는 **저장소·문서에 어떻게 반영할지**(기술)를 실행한다. 두 역할은 겹치지 않는다.

---

## 작업 흐름

```text
CEO
↓
Director(클챗) — 시나리오 / Shot / 프롬프트 / 생성 제작
↓
PM(코비) — 내용 정리 · CEO 의견 반영 · 작업지시서 작성
↓
Engineer(클로) — 문서/저장소 반영 · commit · push
↓
CEO 승인
```

---

## 원칙

- PM은 Director의 창작물을 **수정하지 않는다.**
- PM은 CEO의 요구를 Director가 이해하기 쉬운 형태로 **정리해서 전달만** 한다.
- Engineer는 **승인된 내용만** 저장소·문서에 반영한다.

> 창작은 Director. 판단은 CEO. 정리는 PM. 반영은 Engineer.
