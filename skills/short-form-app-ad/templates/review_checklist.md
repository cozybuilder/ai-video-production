# 광고 완성 후 검수 체크리스트 (G13) — short-form-app-ad

> 전 항목 ✅여야 "검수 대기"로 승격된다. 하나라도 ❌면 해당 게이트로 되돌아간다.
> 이 체크리스트 통과 후에도 G14(CEO 승인) 전에는 영상 생성·크레딧 사용·게시를 하지 않는다.

## 사실성

- [ ] 모든 장면·카피·프롬프트가 G2 기능 인벤토리 안의 기능만 사용했다
- [ ] 기본 OFF 기능·심사 대기 플랫폼을 가능한 것처럼 표현하지 않았다 (G3)
- [ ] G12 금지 표현(치료·개선 단정, 보장성, 효능 주장, 비방)이 없다

## 구조

- [ ] 첫 1~3초 훅이 존재하고 선택 사유가 기록되어 있다 (G5)
- [ ] 문제 → 감정 → 해결 → CTA 4단 구조를 따른다 (G6)
- [ ] CTA가 앱 설치 행동 1개로 단일화되어 있다

## 포맷

- [ ] 9:16 세로 기준으로 설계되었다
- [ ] 15초 안이 완결되어 있다 (30초은 확장 매핑만)
- [ ] Reels / Stories / Shorts 공용 사양(안전 영역·상단 UI 겹침)을 고려했다

## 자막·사운드

- [ ] 자막이 화면당 1메시지이고 음소거 시청만으로 이해된다 (G9)
- [ ] 내레이션·효과음이 장면별로 지정되어 있다

## 프롬프트

- [ ] Veo / Flow / Higgsfield 프롬프트가 별도 블록으로 분리되어 있다 (G10)
- [ ] 프롬프트에도 미구현 기능·금지 표현이 없다
