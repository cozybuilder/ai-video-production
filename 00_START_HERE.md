# 00_START_HERE — 가장 먼저 읽으세요

이 문서는 ai-video-production 저장소에서 **가장 먼저 읽어야 하는 문서**입니다.
새로운 AI(클챗/클로), 새로운 세션, 새로운 협업자는 작업 전에 반드시 이 문서를 읽습니다.

---

## 1. SSOT 원칙 (Single Source of Truth)

- **이 GitHub 저장소가 AI 영상제작의 유일한 공식 기준입니다.**
- 채팅에서의 합의는 **임시 상태**입니다.
- GitHub에 commit / push 되어 반영된 문서만이 **공식 기준**입니다.
- 문서와 기억(또는 채팅 내용)이 충돌하면 **문서가 이깁니다.**

## 2. Document First 원칙

> 결정했으면 문서화한다. 문서화하지 않았으면 결정하지 않은 것이다.

- 어떤 원칙·규칙·결정도 **문서에 반영되기 전까지는 공식이 아닙니다.**
- "말로 합의했다"는 근거가 될 수 없습니다.
- 작업의 완료는 "했다"가 아니라 **"문서로 저장하고 push 했다"** 입니다.

## 3. 클챗과 클로 구분 (매우 중요)

| 구분 | 클챗 (Claude 챗봇) | 클로 (Claude Code) |
|------|--------------------|--------------------|
| 역할 | 분석 · 프롬프트 · 문서 **초안** | 실제 **파일 수정 · commit · push** |
| 결과물 | 텍스트 제안 | 저장소에 반영된 파일 |
| 완료 | 초안 전달 시점 | push 완료 시점 |

> ⚠️ AI 영상제작에서 **클챗과 클로를 절대 혼동하지 않습니다.**
> 초안은 클챗이 쓰고, 실제 저장은 클로만 합니다.

## 4. HARIN 제작 철학

> **하린은 변하지 않습니다. 세상이 변합니다.**

- **고정축 (HARIN):** 얼굴 · 의상 · 헤어 · 실루엣 · 정체성 — 절대 불변
- **변화축 (WORLD):** 배경 · 카메라 · 자연현상 · 세계 · 조명 · 사운드 — 자유롭게 변화

자세한 규칙: [docs/characters/harin/identity_rules.md](docs/characters/harin/identity_rules.md)

## 5. 이미지 승인 전 영상 생성 금지

- **키프레임/이미지가 승인되기 전에는 영상을 생성하지 않습니다.**
- 승인 없는 크레딧(유료 생성) 사용 금지.
- **프롬프트보다 승인된 이미지가 우선**합니다.
- 한 클립 = 한 이벤트.
- 제작 순서: ① 썸네일 프레임 → ② 첫 1초 훅 → ③ 본편 → ④ 사운드 → ⑤ 업로드

## 6. 완료의 정의

작업은 아래가 **모두** 충족될 때만 완료입니다.

1. 코비 지시
2. 사장님 전달
3. 클로 실제 파일 수정
4. commit
5. push
6. 완료보고
7. 코비 검수

> **미응답 = 미완료.** commit/push 실패 시 완료로 보고하지 않습니다.
